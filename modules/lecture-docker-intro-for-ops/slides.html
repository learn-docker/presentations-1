Docker Intro for Ops and Docker Datacenter


What should Operations know about Docker?

  • VM vs Container
  • Ephemeral
  • Stateless vs stateful (app architecture)
  • Orders of magnitude
  • Size and density of storage
  • Speed
  • Immutable Infrastructure
  • talk about differences between containers and VMs. As IT ops, you are probably used to managing VMs. But what are the implications of managing containers along with those VMs.
  • VM’s are more infrastructure whereas containers are more applications. Each container is an app or part of an app
  • Ephemeral - with containers you can easily start and stop them. Can recreate the image and start the container again and its much faster than VM
  • Stateless vs Stateful - most containers are stateless. Stateful parts of applications are separated into different containers. this is part of the whole microservices architecture for applications. Compared with VMs, where you have both the application and state in the VM
  • Magnitude - You would have more containers compared to VMs. 10 to 1? Unlike VM’s where you can monitor each one, there could be far too many containers for 1 on 1 management. It’s more important to understand the application level management as opposed to individual containers. - All about microservices
  • Size - smaller footprint, lower start times
  • Immutable infrastructure - With VMs, if its running in production you can still make changes to the application in the VM, but you need to have the right tools setup. With containers, once it is up and running you generally don’t change things in the container. If the application needs to be re-configured, the apps team will re-build it and as IT ops, you would just re-deploy the container in the environment.
