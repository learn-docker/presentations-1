Mini-lecture:
Organizations and Teams

Public repositories

  • Public repositories created under a user account
    • Openly available
    • Can pull images with authentication
    • Visible to all users of DTR repository dashboard
    • Only the repository author and DTR admins are allowed to push images
  • Public organization repositories
    • Anyone can pull images
    • Visible to all users of DTR repository dashboard
    • Teams with read-write access can push images

Private repositories

  • Private repositories created under a user account
    • Only accessibile to the repository author and DTR admins
  • Private organization repositories
    • Only accessibile to certain teams within the organization
    • Teams can be given read-only or read-write or admin access to the repository.
    • Repository will not appear on DTR dashboard if a user does not have access to it
    • Users trying to push or pull images from it will get an Error, not found message.

Organizations

  • Allows users to be grouped
  • Organization name is used for repositories belonging to the organization

Teams

  • Allows grouping of members within an organization
  • Fine grained permission control for organization repositories

Configuring team members

  • Click the Add user button
  • Select users from the list and click save
  • Can configure membership level afterwards
    • member
    • admin
  • A user can belong to multiple teams and multiple organizations.
A team admin can add other people to the team. However this has to be done through the API. It is not available in the UI at this stage

Adding repositories to a team

  • Repositories must be added to a team before permissions can be configured
  • Can create a new repository, which will be automatically associated with the organization

Repository permissions

  • Read only
    • Can browse and repository and pull images
  • Read write
    • Can do everything from the read only permission
    • Push images
    • Delete tags
  • Admin
    • Can do everything from read only and read write permissions
    • Edit repository descriptiom
    • Set public or private
    • Change team access level

Organization members

  • Users can belong to an organization without belonging to a team
  • Team members are automatically organization members
  • Organization members can:
    • View other members
    • View organization teams and their members
    • View and pull images from public repositories in the organization
  • Organization members do not have the ability to push images to a repository. They must belong to a team with the right permission

Organization admins

  • Individual organization members can be made into an organization admin
  • Organization admins have full admin access to all repositories in that organization
  • Can add users to the organization
  • Can create and configure teams in the organization
