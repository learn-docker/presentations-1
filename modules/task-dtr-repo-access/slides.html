Task: Testing Repository Access

Step 1 - Create private repository

1. Create a private repository for the engineering organization called core-app
2. Configure both the database and web teams to have read only access to the core-app repo
3. Tag and push a hello-world image into the repository. Use the tag 1.0
$ docker tag hello-world /engineering/core-app:1.0
$ docker push /engineering/core-app:1.0

Step 2 - Test access

1. Login to the DTR web UI as Chloe, then barry, and then John. What do you notice with the list of repositories?
2. Login to the UCP web UI as John and pull the /engineering/core-app:1.0 image. What do you notice?

Step 3 - Promote to organization admin

1. Login to DTR web UI as the admin.
2. Promote the user John to become an administrator in the engineering organization
3. Now login to DTR as John. What can you observe in the list of repositories
4. Switch back to UCP (you should be logged in as John).
5. Pull the /engineering/core-app:1.0 image.
